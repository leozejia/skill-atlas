# Skill-Atlas 批判性评审框架

## 核心思维方式

### 1. 批判性思维 (Critical Thinking)
**质疑一切假设，验证所有主张**

- **来源质疑**: 这个 skill 来自哪里？作者可信吗？
- **功能质疑**: 它声称能解决什么问题？真的有效吗？
- **替代方案**: 是否有更好的实现方式？是否重复造轮子？
- **隐藏成本**: 使用它的隐性成本是什么？（学习曲线、依赖、维护）

### 2. 系统思维 (Systems Thinking)
**看整体，不只是局部**

- **生态位**: 这个 skill 在 Nexus 生态中处于什么位置？
- **依赖关系**: 它依赖什么？什么会依赖它？
- **连锁反应**: 引入它会影响其他 skills 吗？
- **长期演化**: 6个月、1年后，这个 skill 还会相关吗？

### 3. 第一性原理 (First Principles)
**回到根本问题**

- **本质问题**: 我们为什么需要这个 skill？根本问题是什么？
- **必要性**: 没有它，我们能用什么更简单的方式解决？
- **最小化**: 它的最小有效版本是什么？

### 4. 反脆弱思维 (Antifragile)
**不仅稳健，还要从波动中受益**

- **鲁棒性**: 如果上游 repo 停止维护，我们怎么办？
- **可替换性**: 容易被替换吗？还是会导致供应商锁定？
- **适应性**: 技术栈变化时，它还能工作吗？

## 评审流程

```
1. 批判性质疑（挑战所有假设）
2. 系统分析（看整体影响）
3. 第一性验证（回到根本）
4. 反脆弱评估（长期稳健性）
5. 综合决策
```

## 决策矩阵

| 维度 | 权重 | 评分 |
|------|------|------|
| 批判性质疑通过率 | 30% | 0-10 |
| 系统性适配度 | 25% | 0-10 |
| 第一性必要性 | 25% | 0-10 |
| 反脆弱性 | 20% | 0-10 |

**总分 >= 7**: distribute
**总分 4-6**: keep
**总分 < 4**: remove
